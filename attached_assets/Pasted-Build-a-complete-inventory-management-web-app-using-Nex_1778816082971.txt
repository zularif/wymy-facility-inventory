Build a complete inventory management web app using Next.js, Supabase, and Tailwind CSS.

I have zero coding experience, so make the app simple to manage and easy to deploy to Vercel.

Use:
1. Next.js for the web app
2. Supabase for database and authentication
3. Tailwind CSS for UI
4. Company OneDrive photo links for item photos
5. Excel only for import and export
6. QR code for technician stock out
7. PDF export for rack labels

Important:
Do not store photos inside the database.
Do not store photos inside Excel.
Store only photo_url in the items table.
The actual item photos will be stored in company OneDrive.

Main modules:
1. Login
2. Dashboard
3. Item Master
4. Stock In
5. Stock Out
6. QR Code Stock Out
7. Current Balance
8. Low Stock Alert
9. Reports
10. Excel Import
11. Excel Export
12. Rack Label Export
13. Audit Log
14. User Roles

User roles:
1. Admin
   - can access everything
2. Storekeeper
   - can manage item master
   - can stock in
   - can stock out
   - can export reports
   - can export labels
3. Technician
   - can scan QR
   - can submit stock out
   - can view own stock out history
   - cannot edit item master
   - cannot export labels
4. Viewer
   - can view dashboard and reports only

Item Master requirements:
1. Add item
2. Edit item
3. Deactivate item
4. Search item
5. Filter by category
6. Filter by location
7. Filter by status
8. Show item photo button if photo_url exists
9. Show current stock
10. Show stock status:

   - Low Stock
   - Out of Stock

Stock In requirements:
1. Select item
2. Enter quantity
3. Enter date
4. Enter reference number
5. Enter supplier
6. Enter remarks
7. Save as movement_type = IN
8. Create audit log

Stock Out requirements:
1. Select item
2. Enter quantity
3. Enter date
4. Enter reference number
5. Enter department
6. Enter requested_by
7. Enter issued_by
8. Enter purpose
9. Enter remarks
10. Save as movement_type = OUT
11. Validate quantity cannot exceed current stock
12. Create audit log

QR Code Stock Out requirements:
1. Each item should have a QR code.
2. QR code should open:
   /stock-out?item_code=ITEM_CODE
3. When technician scans QR code, stock out form opens.
4. Item details should auto load from item_code in URL.
5. If user is not logged in, redirect to login first and return to stock out form after login.
6. Technician only needs to enter:
   - quantity
   - department
   - purpose
   - remarks
7. Date, time, and user should be automatic.
8. Validate quantity cannot exceed current stock.
9. Create stock movement with movement_type = OUT.
10. Create audit log.

Current balance calculation:
Current Stock = opening_stock + total IN - total OUT

Low stock rule:
1. If current stock is 0 or below, show Out of Stock.
2. If current stock is below min_stock, show Low Stock.
3. Otherwise show OK.

Rack Label Export requirements:
Create a Labels page in the sidebar.

The Labels page must allow user to:
1. Search items
2. Filter by category
3. Filter by location
4. Filter by status
5. Select one item, multiple items, or all active items
6. Preview labels
7. Print labels
8. Export labels to PDF

PDF label layout:
1. A4 portrait
2. 8 labels per page
3. 2 columns x 4 rows
4. Each label around 95mm width and 65mm height
5. Page margin around 8mm
6. Gap around 4mm
7. Clean black border around each label
8. Make sure labels do not cut when printing

Each rack label must include:
1. Item code or item ID
2. Item name
3. Spec
4. Category
5. Minimum order
6. Location
7. QR code
8. Item photo from photo_url

Rack label QR code:
1. Generate QR code from:
   /stock-out?item_code=ITEM_CODE
2. QR code size around 25mm x 25mm
3. When scanned, it opens the stock out form with item auto selected.

Rack label photo:
1. Use photo_url from items table.
2. Show photo on label if it can load.
3. If photo cannot load, show placeholder image.
4. If OneDrive image cannot be embedded into PDF because of CORS or direct image issue, show placeholder image in PDF.
